Ready to Run Windows Binaries (Updated for 0.9.2)

Note that windows binaries will be built sometime after the official source code release so expect a lag of a couple of days before the most recent version will show up here.
  • Download the fgfs-win32-msvc-bin-0.9.2.zip and fgfs-base-0.9.2.zip files.
  • Unzip these archives using something like pkzip or winzip. They should both be unzipped into the same place. A "FlightGear" folder will be created.
  • Go to the newly created FlightGear folder and run the "runfgfs.bat" file.
  • The base package includes the same pdf manual that is listed at the top of this page. The manual is 109 pages and quite extensive. You are encouraged to read it first if you have any questions or problems. :-)
  • The MSVC Win32 build is provided courtesy of Frederic Bouvier.
  • Norman Vine provides an alternate Win32 binary compiled with Mingw gcc. Note that this just replaces the MSVC binary package, you must also get the shared base package listed above.

Additional Scenery (Optional but Recommended)

You can download additional scenery if you like. The base package comes with a small scenery area so you can get up and running, but optionally you may download scenery for any portion of the world.
  • Download sections of the world: [ Graphical Interface ] [ FTP Interface ]. You can download scenery for much of the world in 10x10 degree chunks. The files are numbered by the coordinates of their lower, left hand corner. If you move your cursor over a tile on the image, your browser should display the name, size, and last modification date of the section.
  • There is a new utility available (as part of the FlightGear src) called "terrasync". This utility runs in the background in a separate process, monitors your position, and downloads (or updates) the latest greatest scenery from the master scenery server "just in time". There is a README with the terrasync utility that explains how to run it.
